Why ZenTerm

SSH クライアントより静かで、
運用画面より身軽。

ZenTerm は「接続できること」だけでなく、「作業を続けやすいこと」を重視した LP に組み直しました。自分のサーバーを、自分の手触りのまま持ち歩くための設計です。

クラウドなしでつながる

アカウント登録も中継サーバーも不要。ZenTerm はあなたのサーバーに直接つながり、データはそのまま自宅や VPS 側に残ります。

作業が途中で切れない

tmux セッションでタスクを保持し、再接続後も作業が残る設計。AI エージェントの実行や長いビルドとも相性が良い構成です。

導入が短く、迷いにくい

ワンコマンドで Gateway を起動して、表示された QR を読むだけ。Raspberry Pi でも Mac mini でも、最初の一歩をできるだけ短くしています。

3-Step Setup
Features

必要なものを、
一枚の操作面へ。

「ただつながるだけ」ではなく、実際の運用で頻繁に触る操作を 1 か所にまとめています。ターミナル、セッション、ファイル、監視、ペアリング、ブラウザアクセスまでを連続した体験にしました。

Terminal

フルターミナル

xterm.js ベースの本格描画。CJK / Unicode 対応、5,000 行スクロールバックで、軽い確認からしっかりした作業まで扱えます。

Sessions

tmux セッション管理

セッションの作成・切り替え・削除を UI から操作。接続が切れても作業を保ち、複数のタスクを並行して持ち歩けます。

Files

ファイルブラウザ

リモートのディレクトリを閲覧し、テキスト編集やアップロードも GUI で完結。ターミナルと行き来しやすい導線にしています。

Status

システム監視

CPU、メモリ、ディスク、温度、稼働時間を確認。外出先からの軽いヘルスチェックやホームサーバーの見守りにも向いています。

Pairing

QR ペアリング

Gateway 起動時に表示される QR をスキャンするだけで接続情報を入力。トークンや URL の手打ちを最小限に抑えます。

Browser

ブラウザでも開ける

モバイルアプリが手元にない場面でも、PC ブラウザからそのままアクセス可能。運用先や端末を選ばず同じ Gateway を使えます。

Get Started

3 ステップで、
自分のサーバーへ戻れるようにする。

導入手順は短く、日々の運用では迷いにくく。LP からそのまま始められるように、前提条件と起動後の流れも 1 画面にまとめました。

Node.js 20+ tmux macOS / Linux systemd / launchd
1

Gateway を起動

サーバー側で実行すると、初回セットアップからサービス登録まで案内されます。Mac mini、Raspberry Pi、Linux VPS などで動かせます。

$ npx zenterm-gateway
2

QR を読む

起動直後のターミナルに表示される QR コードをアプリでスキャン。URL とトークンが自動で入り、接続時の迷いを減らします。

3

どこからでも戻る

iPhone、iPad、またはブラウザから同じ Gateway にアクセス。セッションやファイル、状態確認をひと続きの操作として扱えます。

Server side

前提は Node.js と tmux。Raspberry Pi 5 のようなホームサーバーから、Mac mini、Linux マシン、VPS まで同じ流れで導入できます。

After setup

ターミナル、セッション、ファイル、システム状態をまとめて触れるので、「確認のために SSH、編集のために別ツール」と分断しなくて済みます。

Ready

次のセッションを、
机の外へ持ち出す。

ZenTerm は、いま使っているサーバーから始められます。まずは Gateway を起動して、最初の QR を読んでください。
